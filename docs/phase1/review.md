# Phase 1 コードレビュー - 将来の改善点

## Lexer パッケージ

### 1. 文字列のエスケープシーケンス未対応
```go
// 現在: "hello\"world" は正しく解析できない
// 将来的に \n, \t, \" などのサポートが必要
```

### 2. Unicode非対応
```go
ch byte // 現在はASCIIのみ
// 日本語変数名などを使う場合は rune に変更が必要
```

### 3. エラー位置情報がない
- 行番号・列番号をTokenに持たせると、エラーメッセージが親切になる
- Phase 1では必須ではないが、将来的に検討

### 4. 負の数のリテラル
- `-10` は現在 `MINUS` + `NUMBER` として解析される
- 仕様上これで正しいが、明示的なテストがあると良い

### 5. テストのエッジケース追加
- 空入力
- 未終端の文字列 `"hello`
- 未終端の複数行コメント `//-- ...`
